# Moltbook-Clone

An automated evaluation framework that simulates conversations between LLMs and SLMs in the **skincare domain**, scores the responses using a keyword-weighted formula, and displays everything in a forum-style UI called **Moltbook**.

Academic project — Computational Science, Merito Gdańsk University (Semester 2).

---

## Architecture

| Module | File | Description |
|--------|------|-------------|
| 1 — Interaction Engine | `interaction_engine.py` | LLM generates a skincare question; SLM answers it |
| 2 — Evaluation Pipeline | `evaluation.py` | Scores answers via `Score(R) = Σ(wi·match) + C(R)` |
| 3 — Frontend | `frontend/` | Static forum-style UI, dark mode, keyword highlighting |

### Scoring formula

```
Score(R) = Σ(wi * match(R, ki)) + C(R)

  ki  = skincare keyword from knowledge base (25 keywords)
  wi  = weight per keyword (default 1.0)
  C(R) = +5 if len(R) >= τ (default 200 chars), else -5
```

## Setup

```bash
# 1. Clone / enter the project directory
cd moltbook-clone

# 2. Install dependencies
pip install -r requirements.txt

# 3. Add your Anthropic API key
cp .env.example .env
# Edit .env and replace "your-key-here" with your actual key
```

## Running

### Dry run (no API calls — uses sample data)

```bash
python main.py --dry-run
```

This loads `sample_output.json`, copies the data into `frontend/data/`, and prints a summary. Use this to demo the frontend without spending tokens.

### Full run (calls the Anthropic API)

```bash
python main.py
```

### Options

```
python main.py --help

  --dry-run        Use sample_output.json instead of calling the API
  --threads N      Number of conversation threads to generate (default: 5)
```

## Viewing Results

After running `main.py` (or `--dry-run`), open the frontend:

```bash
open frontend/index.html        # macOS
# or just double-click frontend/index.html in your file manager
```

The page loads `frontend/data/data.js` (a JS file written by `main.py`) which works without a local server. If you prefer to serve it:

```bash
cd frontend && python3 -m http.server 8080
# then open http://localhost:8080
```

## Configuration

Edit `config.py` to change:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `QUESTION_MODEL` | `claude-sonnet-4-20250514` | Model that generates questions |
| `ANSWER_MODEL` | `claude-haiku-4-5-20251001` | Model that generates answers |
| `NUM_THREADS` | `5` | Number of threads per run |
| `TAU` | `200` | Minimum response length for +5 consistency bonus |
| `MAX_RETRIES` | `3` | API retry limit |
| `SKINCARE_KEYWORDS` | 25 keywords | Knowledge base for scoring |
| `KEYWORD_WEIGHTS` | `1.0` each | Per-keyword scoring weight |
